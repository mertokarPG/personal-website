## Problem

Three new lines, all with different buffers feeding each other. Operators could see current levels but not trajectory — by the time a buffer was empty, the downstream line was already starved.

## Approach

1. Tagged every buffer across all three lines. Pulled historian data into Python.
2. Short-horizon forecast (30–90s) using simple rate-of-change; anything fancier lost to latency.
3. Power BI dashboard with traffic-light forecast tiles — operators need glance-ability, not regression coefficients.
4. Iterated on alert thresholds with shift leads; fewer, truer alerts beat more, noisier ones.
