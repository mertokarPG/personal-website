## Problem

Operators were informed by tribal knowledge. When a line stopped, nobody got a notification; someone just… noticed. Corporate wanted a connected-worker platform; nobody agreed on the shape.

## Approach

1. Started from the message, not the platform: what does an operator need to see, when, with what context.
2. Mapped every PLC tag we'd need, then drew the pipe: PLC → MQTT → Site Service Bus → consumers (MES, Control Tower, notifications).
3. Wrote the integration spec as contracts, not code. When the vendor/build direction flipped twice, the contracts held.
4. Pushed hard for idempotent, replayable events. Operators aren't the only consumer — reporting needs the same stream.
